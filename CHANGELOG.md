# Changelog

ClaudeBox 的所有版本更新都记录在这里。

格式基于 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)。

## [0.5.13] - 2026-05-28

### 新增
- 自动更新新增阿里云 OSS CDN 作为第三个 endpoint:GitHub 与 Cloudflare Worker 都不可达时仍能拉到更新元数据(国内用户兜底)
- 升级失败时 Sidebar 与设置面板新增「打开系统代理设置」按钮(macOS / Windows 各跳到对应系统面板),补齐网络错误引导文案
- `./build.sh oss-mirror [tag]` 独立子命令:从 GitHub Release 下载产物并镜像到阿里云 OSS。OSS 凭证通过 `.oss-publish.json`(已 gitignore)在本地配置,完全不进入 CI/Action

### 优化
- Git Diff 对话框大幅提速:每个文件块拆为 `React.memo` 子组件,行号计算缓存,滚动监听换成 `IntersectionObserver`,行级 `content-visibility: auto` 让浏览器跳过视口外行的 paint/layout——上千行 diff 滚动几乎零卡顿
- 窗口最小宽度从 800 提到 1000,底部工具条改为不换行 + 横向溢出,极窄窗口下输入区不再被挤换行
- 侧边栏会话右键菜单弹出后超出 viewport 时自动反向贴边,避免「删除」等末位选项被屏幕底部裁掉

### 修复
- 任务列表中间状态无法实时刷新:`TaskCreate` 在 tool_use 阶段用 `Date.now()` 当本地 id,与服务端真实 `#N` 不匹配,所有 `TaskUpdate` 静默丢失,直到流结束 `markAllCompleted` 一次性变绿。改为先用 `tool_use_id` 占位,`tool_result` 到达时升级为真实 ID。该问题在 Claude Code CLI 内置 todo 工具从 `TodoWrite` 切到 `TaskCreate`/`TaskUpdate` 后才暴露

## [0.5.12] - 2026-05-21

### 新增
- 顶栏分支徽章合并 diff 统计：项目有未提交改动时分支名前显示 `+N -M`，点击直接打开全局 Diff 对话框；下拉分支列表里当前分支也带 +/-
- 历史用户消息支持「重新输入」：一键把内容回填到输入框，便于改写后再发

### 优化
- Git Diff 对话框升级：左侧文件树（目录折叠 + 单子目录链合并 + 滚动联动高亮）、双列行号 gutter、自动换行开关
- 深色主题重做：从紫蓝调改为中性墨灰，加大 hover / 选中色差，文字对比度增强；blockquote 改用 `color-mix` 跟随主题色
- 输入区瘦身：分支切换器从底部 toolbar 移除（已整合到顶栏徽章），布局更紧凑
- 补齐后端 `git_diff_stat` 命令，分支徽章首次能正确展示 +/- 数

## [0.5.11] - 2026-05-15

### 新增
- 输入框消息队列：任务运行中可继续输入、点击「+」按钮加入队列，当前任务结束后按顺序自动派发；队列条目可单独删除，停止任务时一起清空
- 会话级输入草稿：切走会话时自动保存输入框内容和附件，回到该会话时恢复，避免切来切去丢稿
- 全局 Git Diff 对话框：会话右键菜单 → 「查看代码 diff」，统一查看项目所有未提交改动（含 staged / unstaged，按文件分组，左侧目录树跳转）
- 飞书消息特殊卡片：从飞书端转发过来的对话在聊天流里显示一张蓝色「该消息来自飞书端」标记卡，避免和手动输入混淆
- Windows 标题栏显示「ClaudeBox」应用名（macOS 保持简洁不变）

### 优化
- 会话列表运行中指示器：从静态竖条 + 脉冲改为左侧 accent 竖条 + 渐变扫光动画，状态更醒目
- Edit 工具卡片的 diff 视图：改用等宽字体、增大行高与最大高度、Tab 缩进保留，观感更接近 IDE
- 飞书机器人通知卡片：换用 Lark 原生 `markdown` 元素，表格自动包成代码块以保留列对齐，任务列表 `[x]/[ ]` 转为 ✅/☐ emoji，跨终端渲染更稳定
- 飞书端转发的对话现在按对应模型解析 apiKey / baseUrl，与手动启动一致（之前只读全局配置）
- 会话卡片移除悬浮删除按钮，删除统一收到右键菜单，列表更干净

### 修复
- 修复消息队列自动派发后，前一个子进程残留的 `done` 事件会把新进程的 streaming 状态清掉的竞争：`handleStreamDone` 在非 force 且当前仍在 streaming 时会忽略晚到的 done
- 修复 sidecar 退出清理时可能误删后继进程 stdin handle 的问题：`stdin_handles` 改为存 `(pid, ChildStdin)`，清理前校验 pid 一致

## [0.5.10] - 2026-05-14

### 新增
- 侧边栏搜索框：按项目名 / 路径模糊匹配，搜索时自动展开全部分组
- 「添加项目」按钮收为搜索框右侧的 + 图标，更紧凑
- 会话置顶 + 拖动排序：右键「置顶 / 取消置顶」即可固定到顶部分组，置顶区可拖动调整顺序
- 「置顶 / 最近会话」分组标题，可点击折叠
- 未读黄点：会话在后台完成时打小黄点，再次打开自动清除

### 优化
- 最近会话区按 `updatedAt` 自动倒序，发消息的会话自动上浮（置顶区不受影响，保持手动顺序）
- 运行中指示器从项目名右侧的绿色圆点改为卡片左侧的 accent 色竖向 bar，更不喧宾夺主
- 标题栏去掉「ClaudeBox」字样，视觉更干净

### 修复
- 飞书机器人在 per-model apiKey 场景下回复 “未配置 API Key”：App 启动自动重连和侧边栏开关启动两条路径仍直接读全局 `settings.apiKey`，未跟随 v0.5.7 的解析方案。改为统一走 `resolveModelCreds()`，与设置弹窗手动启动一致——优先取当前模型自带的 apiKey，没有再回退到全局

## [0.5.9] - 2026-05-11

### 修复
- Windows 勾选工具后每步仍要确认：Claude Code CLI 在 Windows 上把 shell 工具注册为 `PowerShell`（Unix/macOS 才是 `Bash`），旧版 allowlist 只认 `Bash` 导致 `canUseTool` 每次都走到"询问用户"分支。现在在 sidecar 中把 `Bash` / `PowerShell` 互为别名放行
- 统一权限判定路径：不再把 `allowedTools` 透传给 SDK（Windows `.cmd` + `cmd.exe /c` 链路上 `--allowedTools` 不稳定），改由 `canUseTool` 作为唯一的放行源，跨平台行为一致

## [0.5.8] - 2026-05-08

### 新增
- 工具调用产生的图片（如 gpt-image skill 输出的 `saved_paths`）自动作为独立相册块渲染在对话流里，点击调用查看器，与用户上传附件一致
- 检查更新失败时增加「手动下载」入口（设置 → 关于、侧边栏 info popover），一键跳转 GitHub Releases
- Node.js / Claude Code 自动安装失败后，提供命令复制 + 「打开终端 / CMD」按钮，引导手动执行

### 优化
- 多张图片在消息气泡里改为 2 列方格相册（≥5 张时显示「+N」蒙层），与发送时的附件视觉一致
- 图片查看器:
  - 多图模式新增 ←/→ 切换、计数器、底部箭头按钮
  - 长图按宽度自适应展示,普通滚轮直接滚动浏览;Ctrl/Cmd + 滚轮(以及 macOS 双指捏合)才缩放
  - 长图滚动时关闭/切换/缩放等工具栏固定在视口,不再跟着内容滚走
- 调试日志改为独立窗口打开,避免内嵌侧栏挤占主窗口宽度;`Ctrl/Cmd+Shift+D` 快捷键和「查看日志」按钮均改为弹出新窗口
- 仅有图片附件、未填写文字时,发送按钮保持禁用,避免发出无说明的空消息

## [0.5.7] - 2026-04-28

### 修复
- 发送消息时误报"缺少配置：API Key"：用户已在"添加模型"里给模型配好独立的 apiKey 且测试通过，但 handleSend 只校验全局 apiKey 导致拦截。改为用 resolveModelCreds 解析当前会话模型对应的 per-model 凭证，与实际请求路径一致

## [0.5.6] - 2026-04-28

### 新增
- 文档附件：支持直接拖拽/粘贴 PDF、Word、Excel、PowerPoint、TXT 等文档到输入框；并增加视觉展示
- 飞书机器人智能化重构：
  - 改为 tool-use agent，具备 list_projects / view_project / list_tasks / create_task / update_task / set_default_project / recall_memory 等原子工具
  - 每个聊天独立的持久化记忆（对话历史、消息日志、任务执行结果、默认项目），7 天过期，保存在 `~/.claudebox/data/lark-memory.json`
  - 执行完成后结果自动回流到该聊天的记忆，下次对话可追溯
  - 路由模型跟随主设置，不再写死
  - 新增 `/use <项目>` 设置默认项目，`/忘记` 清空当前聊天记忆

### 优化
-- 设置 / 添加模型弹窗：点击遮罩不再误关弹窗（避免填一半被关掉），支持 ESC 关闭
- README 显式说明最低系统要求（macOS 13 Ventura / Windows 10 1803+）

### 修复
- Windows 上"添加模型"测试报 `API returned HTTP`（状态码空）：`check_model_available` 从 curl 改为 reqwest 直连，回避 `cmd /C` 对 `\n` 换行和 `%` 变量符的破坏
- macOS 11.2（Big Sur）打开后 UI 无样式：`minimumSystemVersion` 提高到 13.0，安装时即可拦截不兼容系统
