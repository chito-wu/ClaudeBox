# Changelog

ClaudeBox 的所有版本更新都记录在这里。

格式基于 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)。

## [0.5.10] - 2026-05-14

### 修复
- 飞书机器人在 per-model apiKey 场景下回复 “未配置 API Key”：App 启动自动重连和侧边栏开关启动两条路径仍直接读全局 `settings.apiKey`，未跟随 v0.5.7 的解析方案。改为统一走 `resolveModelCreds()`，与设置弹窗手动启动一致——优先取当前模型自带的 apiKey，没有再回退到全局

## [0.5.9] - 2026-05-11

### 修复
- Windows 勾选工具后每步仍要确认：Claude Code CLI 在 Windows 上把 shell 工具注册为 `PowerShell`（Unix/macOS 才是 `Bash`），旧版 allowlist 只认 `Bash` 导致 `canUseTool` 每次都走到"询问用户"分支。现在在 sidecar 中把 `Bash` / `PowerShell` 互为别名放行
- 统一权限判定路径：不再把 `allowedTools` 透传给 SDK（Windows `.cmd` + `cmd.exe /c` 链路上 `--allowedTools` 不稳定），改由 `canUseTool` 作为唯一的放行源，跨平台行为一致

## [0.5.8] - 2026-05-08

### 新增
- 工具调用产生的图片（如 gpt-image skill 输出的 `saved_paths`）自动作为独立相册块渲染在对话流里，点击调用查看器，与用户上传附件一致
- 检查更新失败时增加「手动下载」入口（设置 → 关于、侧边栏 info popover），一键跳转 GitHub Releases
- Node.js / Claude Code 自动安装失败后，提供命令复制 + 「打开终端 / CMD」按钮，引导手动执行

### 优化
- 多张图片在消息气泡里改为 2 列方格相册（≥5 张时显示「+N」蒙层），与发送时的附件视觉一致
- 图片查看器:
  - 多图模式新增 ←/→ 切换、计数器、底部箭头按钮
  - 长图按宽度自适应展示,普通滚轮直接滚动浏览;Ctrl/Cmd + 滚轮(以及 macOS 双指捏合)才缩放
  - 长图滚动时关闭/切换/缩放等工具栏固定在视口,不再跟着内容滚走
- 调试日志改为独立窗口打开,避免内嵌侧栏挤占主窗口宽度;`Ctrl/Cmd+Shift+D` 快捷键和「查看日志」按钮均改为弹出新窗口
- 仅有图片附件、未填写文字时,发送按钮保持禁用,避免发出无说明的空消息

## [0.5.7] - 2026-04-28

### 修复
- 发送消息时误报"缺少配置：API Key"：用户已在"添加模型"里给模型配好独立的 apiKey 且测试通过，但 handleSend 只校验全局 apiKey 导致拦截。改为用 resolveModelCreds 解析当前会话模型对应的 per-model 凭证，与实际请求路径一致

## [0.5.6] - 2026-04-28

### 新增
- 文档附件：支持直接拖拽/粘贴 PDF、Word、Excel、PowerPoint、TXT 等文档到输入框；并增加视觉展示
- 飞书机器人智能化重构：
  - 改为 tool-use agent，具备 list_projects / view_project / list_tasks / create_task / update_task / set_default_project / recall_memory 等原子工具
  - 每个聊天独立的持久化记忆（对话历史、消息日志、任务执行结果、默认项目），7 天过期，保存在 `~/.claudebox/data/lark-memory.json`
  - 执行完成后结果自动回流到该聊天的记忆，下次对话可追溯
  - 路由模型跟随主设置，不再写死
  - 新增 `/use <项目>` 设置默认项目，`/忘记` 清空当前聊天记忆

### 优化
-- 设置 / 添加模型弹窗：点击遮罩不再误关弹窗（避免填一半被关掉），支持 ESC 关闭
- README 显式说明最低系统要求（macOS 13 Ventura / Windows 10 1803+）

### 修复
- Windows 上"添加模型"测试报 `API returned HTTP`（状态码空）：`check_model_available` 从 curl 改为 reqwest 直连，回避 `cmd /C` 对 `\n` 换行和 `%` 变量符的破坏
- macOS 11.2（Big Sur）打开后 UI 无样式：`minimumSystemVersion` 提高到 13.0，安装时即可拦截不兼容系统

## [0.5.5] - 2026-04-27

### 新增
- Windows 无边框模式：Windows 与 macOS 双端视觉体验统一
- 应用内图片预览：输入框缩略图与消息气泡中的图片单击即可打开查看器，支持滚轮/按钮缩放、拖拽平移、双击 1x↔2x、ESC 关闭、右上角「在系统中打开」
- 输入框区域，支持展示实时运行时间

### 修复
- 文件预览与 Markdown 中外链点击会让整个应用被替换、必须重启才能恢复的严重问题（现统一调系统浏览器打开）

## [0.5.4] - 2026-04-23

### 新增
- 更新日志功能：检查更新时展示新版本说明，升级完成后自动弹窗；点击版本号可查看完整历史
- 添加模型独立弹窗：按厂商选择（Anthropic / 智谱 / Kimi / 通义千问 / MiniMax / 小米 MiMo / 自定义），自动填充 baseUrl；每个模型可独立保存 apiKey 与 baseUrl

### 优化
- 厂商下拉改为自定义样式，取代原生 select
- 切换会话时上下文进度条不再闪烁（按会话缓存 tokens，重访立即显示）
- 检查更新失败时给出友好的网络错误提示，支持展开原始错误与重试
- 本地打包 CHANGELOG.md，不依赖 GitHub 也能查看版本历史

### 修复
- 小米 MiMo 控制台链接更正为 platform.xiaomimimo.com
- 去掉「阿里百炼」品牌，改为「通义千问」；旧配置自动迁移
